План профилактического обслуживания операционной системы

Необходимо установить стратегию резервного копирования, в состав которой входит восстановление данных личных файлов. Для выполнения резервного копирования можно использовать программу архивации Microsoft Backup. Способы использования компьютера, а также требования, установленные в организации, определяют частоту выполнения и тип резервного копирования данных.

В зависимости от потребностей можно выбрать различные типы резервного копирования.

Обычное резервное копирование

Обычное резервное копирование также называют полным резервным копированием. При выполнении обычного резервного копирования все выбранные файлы на диске архивируются на архивном носителе. Эти файлы помечаются как архивированные с помощью очистки атрибута архивации.

Копирующая архивация

При копирующей архивации создаются копии всех выбранных файлов. При этом файлы не помечаются как архивированные.

Инкрементное резервное копирование

При инкрементном резервном копировании архивируются только файлы и папки, созданные или измененные после последнего полного или инкрементного резервного копирования. Эти файлы помечаются как архивированные с помощью очистки атрибута архивации. В результате выполнения этого действия начальная точка разностного резервного копирования перемещается вперед, при этом нет необходимости повторного архивирования всего содержимого диска. Для восстановления последовательной резервной копии сначала необходимо восстановить последнюю полную резервную копию, после чего восстановить по порядку все последовательные резервные копии.

Разностное (дифференциальное) резервное копирование

При разностном резервном копировании создаются резервные копии всех файлов и папок, которые были созданы или изменены после последнего полного или последовательного резервного копирования. При разностном резервном копировании файлы не помечаются как архивированные. Копии создаются с одной и той же начальной точки до выполнения следующего последовательного или полного резервного копирования. Создание разностных резервных копий играет важную роль, поскольку для восстановления всех данных требуются только последняя полная копия и следующие за ней разностные.

Ежедневное резервное копирование

При выполнении ежедневного копирования создаются резервные копии только тех файлов, которые были изменены в день резервного копирования. При ежедневном копировании атрибут архивации не меняется.

Выполнение резервного копирования может занять много времени. При тщательном соблюдении стратегии резервного копирования нет необходимости постоянно создавать резервные копии всех файлов. Резервное копирование необходимо только для тех файлов, которые были изменены с момента его последнего выполнения.

Для получения доступа к программе архивации в Windows 7 перейдите по следующим пунктам:

Пуск > Панель управления > Архивация и восстановление

Для получения доступа к программе архивации в Windows Vista перейдите по следующим пунктам:

Пуск > Все программы > Стандартные > Служебные > Состояние и настройка архивации

Для получения доступа к программе архивации в ОС Windows XP Professional перейдите по следующим пунктам:

Пуск > Все программы > Стандартные > Служебные > Архивация данных
