Ժամանակակից օպերացիոն համակարգեր

ՕՀ-ի տերմինները և բնութագրերը

Անկախ համակարգչի և ՕՀ-ի չափերից և բարդությունից, բոլոր ՕՀ-ները իրականացնում են հետևյալ 4 հիմնական ֆունկցիաները․

  • Ապարատային ապահովման ղեկավարում (Control hardware access)

  • Ֆայլերի և թղթապանակների ղեկավարում(Manage files and folders)

  • Օգտագործողի ինտերֆեյսի ապահովում (Provide a user interface)

  • Ծրագրային ապահովման կառավարում

Ապարատային ապահովման ղեկավարում

ՕՀ-ն ղեկավարում է ապարատային և ծրագրային ապահովման միջև փոխազդեցությունը: Որպեսզի հնարավոր լինի օգտագործել ապարատային ապահովումը, ՕՀ-ն յուրաքանչյուր սարքի համար տեղադրում է ծրագիր, որը կոչվում է սարքի դրայվեր: Սարքի դրայվերն իրենից ներկայացնում է սարքը արտադրողի կողմից գրված փոքր ծրագիր, որը տրվում է սարքի հետ միասին: Երբ ֆիզիկական սարքը տեղադրվում է, ՕՀ-ն գտնում և տեղադրում է տվյալ սարքի դրայվերը: Դրայվերի տեղադրումը կատարվում է ավտոմատ plug-and-play (PnP) գործընթացի կողմից: Այնուհետև ՕՀ-ն կոնֆիգուրացնում է սարքավորումը և թարմացնում է ռեգիստրը, որն իրենից ներկայացնում է համակարգչի մասին ողջ ինֆորմացիան պարունակող տվյալների բազա (ինֆորմացիա ծրագրային ապահովման, օգտագործողի, ապարատային ապահովման, ցանցային կարգավորումների, ֆայլերի տիպերի մասին):

Եթե ՕՀ-ն չի կարողանում գտնել սարքավորման դրայվերը, այդ դեպքում մասնագետը դրայվերը տեղադրում է ձեռքով` օգտագործելով սարքավորման հետ տրամադրվող կրիչը (միջոցը) կամ բեռնելով այն արտադրողի կայքից:

Ֆայլերի և թղթապանակների ղեկավարում

ՕՀ-ն կոշտ սկավառակի վրա ստեղծում է ֆայլի ստրուկտուրան` տվյալների պահման համար: Ֆայլը դա միմյանց հետ կապված տվյալների բլոկ է, որին տրված է մեկ անուն և դիտարկվում է որպես մեկ ամբողջական միավոր: Ծրագրի և տվյալների ֆայլերը խմբավորված են մեկ դիրեկտորիայի մեջ: Ֆայլերը և դիրեկտորիաները կազմակերպված են տվյալների փնտրումը և օգտագործումը հեշտացնելու համար: Դիրեկտորիաները կարող են գտնվել այլ դիրեկտորիաների ներսում, այդպիսի դիրեկտորիաները կոչվում են ենթադիրեկտորիաներ: Դիրեկտորիաները ՕՀ-ում կոչվում են թղթապանակներ, իսկ ենթադիրեկտորիաները` ենթաթղթապանակներ:

Օգտագործողի ինտերֆեյսի ապահովում

ՕՀ-ն թույլ է տաիս օգտագործողին համագործակցել ապարատային և ծրագրային ապահովման հետ օգտագործողի ինտերֆեյսի միջոցով: Գոյություն ունեն երկու տիպի ինտերֆեյս`

  • Հրամանային տողով ինտերֆեյս (Command Line Interface – CLI), որի դեպքում օգտագործողը մուտքագրում է հրամաններ: CLI-ն ունի Նկ.1-ում բերված տեսքը։

  • Օգտագործողի գրաֆիկական ինտերֆեյս (Graphical User Interface -GUI): Այս ինտերֆեյսի դեպքում օգտագործում է մենյուներ և պատկերներ(icon): Նկ.2

Ծրագրային ապահովման կառավարում

ՕՀ-ը գտնում է աշխատածրագրերը և բեռնավորում օպերատիվ հիշողության մեջ: աշխատածրագրերը իրենից ներկայացնում է ծրագրեր ինչպիսիք են տեքստային խմբագրիչները (word, open office), էլեկտրոնային աղյուսակների խմբագրիչներ (Excel, open office), խաղերը և բազմաթիվ այլ ծրագրեր:

API (Application Programming Interface) իրենից ներկայացնում է ծրագրավորողների կողմից օգտագործվող ուղեցույց, որի շնորհիվ երաշխավորվում է, որ նրանց կողմից նախագծվող, ստեղծվող ծրագիրը համատեղելի է օպերացիոն համակարգի հետ: Ստորև ներկայացված են API-ի մի քանի օրինակներ`
