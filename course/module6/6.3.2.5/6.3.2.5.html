Адресация оборудования в сети

Если к локальной сети подключено большое число компьютеров, настройка IP-адресов для каждого узла вручную может занимать много времени и часто связана с ошибками. Сервер DHCP автоматически назначает IP-адреса, что упрощает процесс адресации. Автоматическая настройка TCP/IP также снижает риск назначения дублированных или недопустимых IP-адресов.

Сервер DHCP поддерживает список IP-адресов, которые можно назначить, и управляет процессом их назначения, поэтому каждое устройство в сети получает уникальный IP-адрес. После получения запроса от узла сервер DHCP выбирает адресную информацию IP из набора предопределенных адресов, хранящихся в базе данных. После того как адресная информация IP выбрана, сервер DHCP предоставляет эти значения запрашивающему узлу в сети. Если узел принимает предложение, сервер DHCP назначает данный IP-адрес на определенный период времени. Эта процедура называется арендой. По истечении срока аренды сервер DHCP может использовать этот адрес для другого компьютера, добавленного в сеть. Кроме того, устройство может возобновить аренду для сохранения данного IP-адреса.

Чтобы компьютер в сети мог воспользоваться услугами сервера DHCP, он должен быть способен обнаружить этот сервер в локальной сети. Компьютер можно настроить так, чтобы он принимал IP-адреса от сервера DHCP, выбрав вариант Получать IP-адрес автоматически в окне настройки сетевого адаптера, как показано на рисунке 1. После того как компьютер будет настроен для автоматического получения IP-адресов, все другие поля настройки IP-адресов станут недоступны. Параметры DHCP настраиваются одинаково для проводной и беспроводной сетевой платы.

Компьютер непрерывно запрашивает IP-адреса от сервера DHCP каждые пять минут. Если компьютеру не удается связаться с сервером DHCP для получения IP-адреса, ОС Windows автоматически назначает локальный IP-адрес канала. Если компьютеру назначен локальный IP-адрес канала (в диапазоне от 169.254.0.0 до 169.254.255.255), этот компьютер сможет обмениваться данными только с компьютерами, подключенными к той же сети в данном диапазоне IP-адресов.

Сервер DHCP автоматически назначает узлу следующие сведения о настройке IP-адреса:

  • IP-адрес

  • Маска подсети

  • Основной шлюз

  • Дополнительные значения, например адрес сервера DNS, как показано на рисунке 2

В ОС Windows 7 перейдите по следующим пунктам:

Пуск > Панель управления > Центр управления сетями и общим доступом > Изменение параметров адаптера > щелкните правой кнопкой мыши Подключение по локальной сети > Свойства > Протокол Интернета версии 4 (TCP/IPv4) > Свойства > выберите положение переключателя Получать IP-адрес автоматически > OК > OК

Настройка альтернативных параметров IP

Установка альтернативной настройки IP в ОС Windows упрощает перемещение между сетями, требующими использования сервера DHCP, и сетями, в которых используются статические параметры IP. Если компьютеру не удается связаться с сервером DHCP в сети, ОС Windows использует альтернативную настройку IP, назначенную сетевой интерфейсной плате. Альтернативная настройка IP также заменяет адрес автоматической IP адресации (APIPA), назначаемый ОС Windows в случаях, когда не удается связаться с сервером DHCP.

Для создания альтернативной настройки IP, как показано на рисунке 3, перейдите на вкладку Альтернативная конфигурация в окне свойств сетевой платы.

DNS

Для получения доступа к серверу DNS компьютер использует IP-адрес, настроенный в параметрах DNS сетевой платы компьютера. Сервер DNS разрешает или сопоставляет имена и URL-адреса узлов IP-адресам.

Все компьютеры Windows содержат кэш DNS, в котором хранятся недавно разрешенные имена узлов. Кэш является первым местом, в котором клиент DNS ищет разрешение имени узла. Поскольку кэш размещен в памяти, разрешенные IP-адреса извлекаются быстрее, чем при использовании сервера DNS, и при этом не создается сетевой трафик.
